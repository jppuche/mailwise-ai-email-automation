# Technical Decisions — mailwise

> Every architectural choice documented with context, alternatives considered, and rationale.

| Date | Decision | Context | Alternatives considered |
|------|----------|---------|------------------------|
| 2026-02-20 | Backend framework: FastAPI | Async native, auto OpenAPI docs, Pydantic v2 validates data contracts at API boundary. Dependency injection maps cleanly to adapter pattern. | Django + DRF (sync by default, admin overlaps dashboard), Flask (more assembly, no auto docs) |
| 2026-02-20 | Database: PostgreSQL | Relational integrity for email→classification→routing FK chains. JSONB for semi-structured data (LLM raw output, routing payloads). pg_trgm for full-text search under 5s over 100K emails. Docker Compose makes setup trivial. | SQLite (no concurrent writes for background workers), MongoDB (overkill for relational data) |
| 2026-02-20 | ORM: SQLAlchemy 2.0 (async) + Alembic | Industry standard. Async via asyncpg driver for FastAPI. Auto-migrations on startup. Type-annotated `Mapped[]` models propagate through mypy. Dual session factories: async (FastAPI) + sync (Celery). | Tortoise ORM (smaller ecosystem), raw asyncpg (no migrations, tedious CRUD) |
| 2026-02-20 | Frontend: React + Vite + TypeScript | SPA required for complex dashboard UI: drag-to-reorder routing rules, side-by-side email/draft review, real-time charts, search/filter. TypeScript enforces data contracts via auto-generated types from OpenAPI. | Jinja2 + HTMX (limited interactivity for drag-reorder/charts), Vue 3 (smaller job market) |
| 2026-02-20 | Task queue: Celery + Redis | Native retry with exponential backoff. Task chaining for email state machine (FETCHED→CLASSIFIED→ROUTED→...). Redis triples as: Celery broker, classification cache, and refresh token store. | asyncio + tenacity (simpler but no persistent queue — tasks lost on restart), arq (lighter but smaller community) |
| 2026-02-20 | Scheduler: APScheduler (dedicated container) | Lightweight cron-like polling at configurable interval. Runs in dedicated container to prevent duplicate polls if API scales horizontally. | Celery Beat (heavier, couples scheduling to Celery), cron (not portable) |
| 2026-02-20 | LLM integration: LiteLLM | Unified `completion()` for OpenAI, Anthropic, and Ollama. Config-level model switch, not code change. Different models per task: cheap/deterministic for classification (temp 0.1), capable/creative for drafts (temp 0.7). | Direct SDKs (two SDKs to maintain), LangChain (over-engineered, heavy dependencies, frequent breaking changes) |
| 2026-02-20 | Email SDK: google-api-python-client | Official Google SDK. OAuth2 with automatic token refresh. Minimum scopes: gmail.readonly + gmail.modify + gmail.compose. | simplegmail (unmaintained) |
| 2026-02-20 | Notification SDK: slack-sdk (official) | AsyncWebClient for FastAPI. Block Kit formatting for structured routing notifications. Built-in rate limit handling. | slack-bolt (overkill for outbound-only notifications), httpx direct (manual auth and rate-limiting) |
| 2026-02-20 | CRM: HubSpot (hubspot-api-client) | Free tier available for demo — reviewers can actually run the system. Official Python SDK covers all required operations. Adapter pattern means Salesforce can be added later without core changes. | Salesforce/simple-salesforce (complex auth, no free tier, SOQL queries) |
| 2026-02-20 | Auth: JWT (python-jose) + bcrypt + Redis | Stateless API auth for React SPA. Short-lived access tokens (15 min) + Redis-stored refresh tokens with configurable TTL. Two roles as JWT claims: Admin, Reviewer. | FastAPI-Users (opinionated for a 2-role system), session cookies (less suited for SPA) |
| 2026-02-20 | Containers: Docker Compose | Single command deployment. 6 services: api, worker, scheduler, db, redis, frontend. Health checks for all services. Dev and prod profiles. | Podman (smaller ecosystem), bare metal (not cloud-ready) |
| 2026-02-20 | Testing: pytest + pytest-asyncio + httpx + factory-boy + Playwright | Unit/integration/E2E coverage. httpx AsyncClient for FastAPI testing. factory-boy for realistic, contract-compliant test data. Playwright for dashboard E2E. | unittest (verbose, no async-native support) |
| 2026-02-20 | Code quality: ruff + mypy | ruff replaces flake8+isort+black in one Rust-based tool. mypy enforces type hints across all adapter boundaries. Both integrated into CI pipeline. | flake8+black+isort (4 tools vs 1), pyright (less mainstream in CI) |
| 2026-02-20 | Project structure: src layout + pyproject.toml | Modern Python packaging standard. `src/` layout prevents accidental imports. `pyproject.toml` consolidates all tool config. `adapters/` directory maps to 4 integration families (email, channel, CRM, LLM). | flat layout (import confusion), setup.py (deprecated) |
| 2026-03-02 | UI/UX: Tailwind CSS v4 + shadcn/ui | Migration from 3,500+ lines of pure CSS to utility-first styling with component library. 20 shadcn/ui components, Lucide icons, OKLCH design tokens, full dark mode support. | Pure CSS improvements (less consistency), Tailwind only without shadcn (less component standardization) |
